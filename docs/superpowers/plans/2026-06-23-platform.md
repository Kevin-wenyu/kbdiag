# 平台层 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立三道安全网——Claude Code 内 git 危险操作拦截、GitHub Actions CI（shellcheck + build + smoke）、本地 shellcheck pre-commit hook。

**Architecture:** git-guardrails 写入项目级 `.claude/settings.json`；CI 走 `.github/workflows/ci.yml`，不跑集成测试（Lima VM 不可达）；pre-commit hook 本地安装，`scripts/setup-dev.sh` 提交到仓库供团队复现。

**Tech Stack:** Pure Bash, GitHub Actions (ubuntu-latest), shellcheck

## Global Constraints

- `.claude/settings.json` 使用项目级路径（`.claude/settings.json`），不写全局
- CI 不跑集成测试（`test/run_tests.sh`），只跑 shellcheck + build + smoke
- `scripts/setup-dev.sh` 必须幂等：重复运行不产生副作用
- `shellcheck lib/*.sh` 必须零错误通过（警告用行内 `# shellcheck disable=SCxxxx` 抑制）
- 改完后 `bash build.sh` 重建 `dist/kbdiag`，验证仍可运行
- 提交信息英文

### Task 1: git-guardrails（Claude Code hook）

**Files:**
- Create/Modify: `.claude/settings.json`

**Interfaces:**
- Produces: Claude Code 内执行危险 git 命令时弹出拦截提示，需要用户手动确认

- [ ] **Step 1: 确认 `.claude/` 目录存在**

```bash
ls .claude/ 2>/dev/null || mkdir -p .claude
```

- [ ] **Step 2: 查看现有 settings.json（如有）**

```bash
cat .claude/settings.json 2>/dev/null || echo "(empty)"
```

- [ ] **Step 3: 写入 git 危险命令拦截配置**

将 `.claude/settings.json` 写为以下内容（如已有其他配置，合并 `hooks` 字段）：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'cmd=$(echo \"$CLAUDE_TOOL_INPUT\" | python3 -c \"import sys,json; d=json.load(sys.stdin); print(d.get(\\\"command\\\",\\\"\\\"))\" 2>/dev/null); if echo \"$cmd\" | grep -qE \"git push.*--force|git push.*-f[^i]|git reset.*--hard|git clean.*-f|git branch.*-D\"; then echo \"BLOCKED: dangerous git command detected: $cmd\" >&2; exit 1; fi'"
          }
        ]
      }
    ]
  }
}
```

- [ ] **Step 4: 验证 JSON 合法**

```bash
python3 -m json.tool .claude/settings.json > /dev/null && echo "JSON valid"
```
预期：`JSON valid`

- [ ] **Step 5: Commit**

```bash
git add .claude/settings.json
git commit -m "feat: add git-guardrails hook to block dangerous git commands in Claude Code"
```

### Task 2: GitHub Actions CI

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Produces: push/PR 到 main 时自动运行 shellcheck + build + smoke，绿色 badge

- [ ] **Step 1: 创建目录**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: 创建 `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  shellcheck:
    name: Shellcheck
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install shellcheck
        run: sudo apt-get install -y shellcheck
      - name: Run shellcheck
        run: shellcheck lib/*.sh

  build:
    name: Build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build dist/kbdiag
        run: bash build.sh
      - name: Verify binary exists and is executable
        run: |
          test -f dist/kbdiag || (echo "dist/kbdiag not found" >&2; exit 1)
          test -x dist/kbdiag || (echo "dist/kbdiag not executable" >&2; exit 1)

  smoke:
    name: Smoke test
    runs-on: ubuntu-latest
    needs: build
    steps:
      - uses: actions/checkout@v4
      - name: Build
        run: bash build.sh
      - name: Version flag
        run: dist/kbdiag --version
      - name: Help flag
        run: dist/kbdiag --help
```

- [ ] **Step 3: 本地验证 YAML 语法（需 python3-yaml）**

```bash
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))" && echo "YAML valid"
```
预期：`YAML valid`

- [ ] **Step 4: 先确认 shellcheck 零错误通过（CI 会执行此步）**

```bash
shellcheck lib/*.sh
```
预期：无输出（零错误）。若有错误，按 Step 5 处理。

- [ ] **Step 5: 修复 shellcheck 错误（若有）**

对每个报错，选择以下之一：
- 修改代码消除错误
- 在报错行上方添加行内抑制（仅用于误报）：`# shellcheck disable=SC2034`

完成后重跑直到零错误：
```bash
shellcheck lib/*.sh
```

- [ ] **Step 6: Commit 并 push，触发 CI**

```bash
git add .github/workflows/ci.yml
git commit -m "feat: add GitHub Actions CI (shellcheck, build, smoke)"
git push
```

- [ ] **Step 7: 确认 CI 通过**

```bash
gh run list --limit 3
```
预期：最新 run 状态为 `completed` / `success`。若失败，查看日志：
```bash
gh run view --log
```

### Task 3: shellcheck pre-commit hook

**Files:**
- Create: `scripts/setup-dev.sh`（提交到仓库）
- 安装目标：`.git/hooks/pre-commit`（本地，不提交）

**Interfaces:**
- Produces: `git commit` 前自动运行 `shellcheck lib/*.sh`；失败则中断提交并显示错误

- [ ] **Step 1: 创建 `scripts/setup-dev.sh`**

```bash
#!/usr/bin/env bash
# setup-dev.sh — install local git hooks for kbdiag development
# Run once after cloning: bash scripts/setup-dev.sh
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
HOOK="$REPO_ROOT/.git/hooks/pre-commit"

cat > "$HOOK" <<'HOOK_EOF'
#!/usr/bin/env bash
# pre-commit: run shellcheck on lib/*.sh before every commit
set -euo pipefail

if ! command -v shellcheck >/dev/null 2>&1; then
  echo "pre-commit: shellcheck not found, skipping (install with: brew install shellcheck)" >&2
  exit 0
fi

REPO_ROOT="$(git rev-parse --show-toplevel)"
echo "pre-commit: running shellcheck on lib/*.sh"
shellcheck "$REPO_ROOT"/lib/*.sh
echo "pre-commit: shellcheck passed"
HOOK_EOF

chmod +x "$HOOK"
echo "Installed pre-commit hook: $HOOK"
```

- [ ] **Step 2: 赋权**

```bash
chmod +x scripts/setup-dev.sh
```

- [ ] **Step 3: 验证脚本语法**

```bash
bash -n scripts/setup-dev.sh && echo "syntax OK"
```
预期：`syntax OK`

- [ ] **Step 4: 安装 hook（本地）**

```bash
bash scripts/setup-dev.sh
```
预期：`Installed pre-commit hook: /path/to/.git/hooks/pre-commit`

- [ ] **Step 5: 验证 hook 可执行**

```bash
ls -la .git/hooks/pre-commit
```
预期：权限含 `x`，文件存在。

- [ ] **Step 6: 触发一次 hook 验证**

```bash
.git/hooks/pre-commit
```
预期：`pre-commit: shellcheck passed`（零错误）

- [ ] **Step 7: Commit**

```bash
git add scripts/setup-dev.sh
git commit -m "feat: add scripts/setup-dev.sh to install shellcheck pre-commit hook"
git push
```
