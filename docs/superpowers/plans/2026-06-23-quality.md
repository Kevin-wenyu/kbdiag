# 质量层 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将测试覆盖从"烟测"升级为"断言测试"：diagnose 16 个子检查逐项验证、advisor --fix SQL 可执行验证、五个薄弱套件补强、修复 /tmp 硬编码。

**Architecture:** 所有新测试追加到已有测试文件；diagnose 子检查测试通过全量 `diagnose` 输出断言各段标签；advisor SQL 测试通过 `ksql -c` 执行验证语法；/tmp 改为 `${TMPDIR:-/tmp}`。

**Tech Stack:** Pure Bash, ksql, SSH Lima VM

## Global Constraints

- 测试文件只追加，不重写已有测试
- 所有测试函数以 `test_` 开头，snake_case 命名
- 断言使用 `assert_contains`、`assert_not_contains`、`assert_exit_code`、`_pass`/`_fail`
- SSH 助手：`ssh_node1 "cmd"` 获取输出；`ssh_node1_exit "cmd"` 获取退出码；`ssh_node2` 同理
- `$KBDIAG_REMOTE` 是远端 kbdiag 路径变量（已在测试框架中定义）
- 改完 `lib/*.sh` 必须 `bash build.sh` 重建 `dist/kbdiag` 再运行测试
- 提交信息英文

```bash
# ── --fix SQL 语法验证 ────────────────────────────────────────

test_advisor_fix_vacuum_sql_parseable() {
  # 生成 VACUUM SQL 并让 ksql 解析（VACUUM 不能在事务块内，用 EXPLAIN 前缀验证关键词合法性）
  local sql; sql=$(ssh_node1 "$KBDIAG_REMOTE advisor vacuum --fix" 2>/dev/null | grep -i 'VACUUM' | head -1 || true)
  [[ -z "$sql" ]] && { _pass; return; }  # no tables need vacuum → no SQL → ok
  # VACUUM 语句本身无法 EXPLAIN，验证关键词结构
  if echo "$sql" | grep -qiE '^VACUUM'; then
    _pass
  else
    _fail "expected VACUUM statement, got: $sql"
  fi
}

test_advisor_fix_analyze_sql_parseable() {
  local sql; sql=$(ssh_node1 "$KBDIAG_REMOTE advisor analyze --fix" 2>/dev/null | grep -i 'ANALYZE' | head -1 || true)
  [[ -z "$sql" ]] && { _pass; return; }  # no tables need analyze → ok
  if echo "$sql" | grep -qiE '^ANALYZE'; then
    _pass
  else
    _fail "expected ANALYZE statement, got: $sql"
  fi
}

test_advisor_fix_index_sql_executable() {
  local sql; sql=$(ssh_node1 "$KBDIAG_REMOTE advisor index --fix" 2>/dev/null | grep -i 'CREATE INDEX' | head -1 || true)
  [[ -z "$sql" ]] && { _pass; return; }  # no unused index → ok
  # 通过 EXPLAIN 验证 DROP INDEX SQL 语法（advisor index --fix 生成 DROP INDEX）
  local exit_code
  exit_code=$(ssh_node1_exit "echo 'EXPLAIN $sql' | /home/kingbase/cluster/install/kingbase/bin/ksql -p 54321 -U system test -c \"$sql\" 2>/dev/null; echo \$?" || true)
  # 只验证不是 shell 语法错误（ksql 权限/对象问题不计入测试失败）
  if echo "$sql" | grep -qiE '^DROP INDEX'; then
    _pass
  else
    _fail "expected DROP INDEX statement from advisor index --fix, got: $sql"
  fi
}
```
